# CONTEXTO: ROBÔ IMPORTADOR DE QUADRINHOS - HQ-HUB

## PROJETO
HQ-Hub é um catálogo de histórias em quadrinhos em português. O objetivo é importar dados do site "Guia dos Quadrinhos" (www.guiadosquadrinhos.com) e armazená-los no banco de dados do aplicativo.

## ROBÔ PRINCIPAL
**Arquivo**: docs/importacao/ferramentas/robo_importador_texto.py
**Função**: Scraper que extrai dados de séries, edições e histórias do Guia dos Quadrinhos e gera arquivos JSON para importação.

## FUNCIONALIDADES E FLAGS

### Flag: --url-inicial (obrigatório)
Define a URL da primeira edição/série a processar.
Exemplo: --url-inicial "http://www.guiadosquadrinhos.com/colecao/colecao-classica-marvel"

### Flag: --quantidade (obrigatório)
Quantidade de edições a processar a partir da URL inicial.
Exemplo: --quantidade 5 (processa 5 edições começando da edição apontada pela URL)

### Flag: --saida (obrigatório)
Pasta onde o arquivo JSON será salvo.
Exemplo: --saida "rascunhos-importacao"

### Flag: --organizar-saida-por-serie (opcional)
Ativa organização automática: cria subpasta com nome da série e nomeia arquivo como "serie-X-a-Y.json"
Sem essa flag: salva tudo em um arquivo único.
Com essa flag: cria estrutura de pastas e arquivos organizados por série.

## COMPORTAMENTO IMPORTANTE

### Limitação de Edições
O robô agora limita edições corretamente a partir da URL inicial:
- Se a URL aponta para edição 6, --quantidade 5 gera JSON com edições 6, 7, 8, 9, 10
- Anteriormente geraria 1, 2, 3, 4, 5 (BUG CORRIGIDO)

### Nomes de Arquivo com --organizar-saida-por-serie
Padrão: "serie-nome-X-a-Y.json"
Exemplo output structure:
```
rascunhos-importacao/
  colecao-classica-marvel/
    colecao-classica-marvel-1-a-5.json
    colecao-classica-marvel-6-a-10.json
    colecao-classica-marvel-21-a-40.json
```

### Evitar Sobrescrita
Se arquivo já existe, o robô adiciona sufixo com timestamp (YYYYMMDD-HHMMSS).
Exemplo: "colecao-classica-marvel-1-a-5_20260704-143022.json"

## BACKEND - IMPORTAÇÃO
**Arquivo**: backend/src/main/java/br/com/hqhub/resource/ImportacaoCatalogoResource.java
**Endpoint**: POST /importacoes/catalogo
**Headers**: Content-Type: application/json
**Funcionalidade**: Recebe JSON e importa no banco de dados.

### Retry Logic (RECENTE)
- Tenta 3 vezes em caso de falha transitória (deadlock, lock timeout)
- Aguarda 300ms → 600ms → 900ms entre tentativas
- Detecta erros transientes por SQL state code (PostgreSQL)

### Organização no Banco
- Procura edição por: número + série
- Se existe: atualiza
- Se não existe: cria nova
- Insere histórias relacionadas

## WORKFLOW TÍPICO
1. Rodar robô com URL inicial + quantidade + --organizar-saida-por-serie
2. Gera JSON estruturado com série/edições/histórias
3. Enviar JSON para backend via importação (POST /importacoes/catalogo)
4. Backend salva dados no banco com retry automático se necessário

## URLS DE CAPA
Atualmente bloqueadas pelo Guia dos Quadrinhos por:
- Anti-hotlink (Referer header checking)
- User-Agent blocking
- Rate limiting
- Cloudflare/WAF

**Solução futura**: Mapeamento de URLs customizado (OneDrive ou local) para sobrescrever URLs do Guia.

## ERROS COMUNS E DIAGNÓSTICO

### Erro: "O sistema não pode encontrar o caminho especificado" ou "No such file or directory"
**Causa**: Comando rodado da pasta errada.
**Como identificar**: O caminho no erro não bate com a estrutura real. Se o erro aponta para `c:\Users\User\...` e o caminho esperado é relativo ao projeto, o `cd` foi esquecido ou errado.
**Solução**: Sempre confirmar com `Get-Location` antes de rodar. Garantir que está em `c:\Users\User\OneDrive\Área de Trabalho\HQ-HUB`.

### Erro: "python : The term 'python' is not recognized"
**Causa**: Python não está no PATH ou o terminal está fora do ambiente correto.
**Solução**: Usar `python3` ou verificar instalação com `Get-Command python`.

### Erro: "Edições: 0 / Histórias: 0" no JSON gerado
**Causa**: O parser não reconheceu a estrutura da página do Guia dos Quadrinhos (ex: edição sem número no título, como volumes únicos/omnibus).
**Consequência**: O passo seguinte (`robo_aplicar_url_capa.py`) falha com `ValueError: O JSON de entrada não possui edições em 'edicoes'`.
**Solução**: O robô foi corrigido para tratar esse caso (fallback para edições sem número). Apagar os arquivos intermediários gerados (`-base-guia.json` e `-capas-panini.txt`) e rodar novamente.

### Erro: "Nao foi possivel identificar o padrao numerico da URL inicial"
**Causa**: A URL da Panini não termina em número (ex: `panini.com.br/aniquilacao` em vez de `panini.com.br/aniquilacao-vol-1`).
**Solução**: O robô foi corrigido para aceitar URLs fixas. Apagar arquivos intermediários e rodar novamente.

### Erro: "ValueError: O JSON de entrada não possui edições em 'edicoes'"
**Causa**: O arquivo `-base-guia.json` foi gerado com `totalEdicoes: 0` (problema no parser).
**Solução**: Ver diagnóstico de "Edições: 0" acima.

### Erro: Comando trava sem saída após iniciar
**Causa**: Rate limiting ou bloqueio temporário do Guia dos Quadrinhos.
**Solução**: Aguardar 1-2 minutos e tentar novamente. Aumentar `--tentativas 3`.

### Erro: Arquivos intermediários corrompidos de tentativa anterior
**Causa**: Uma execução com erro deixou arquivos `-base-guia.json` ou `-capas-panini.txt` incompletos.
**Solução**: Apagar os arquivos intermediários com `Remove-Item` antes de rodar novamente (ver Caso 2 acima).

## TROUBLESHOOTING

**Problema**: Robô para no meio da execução
- Solução: Verificar conexão internet, rate limiting do Guia, ou estrutura mudou no site

**Problema**: "Erro interno do servidor" na importação
- Solução: Backend tem retry automático, tentar novamente em poucos segundos

**Problema**: JSON gerado vazio ou incompleto
- Solução: Verificar se URL inicial é válida no Guia dos Quadrinhos. Testar a URL no navegador.

**Problema**: Arquivos sendo sobrescritos
- Solução: Usar `--organizar-saida-por-serie` para criar nomes únicos + pastas
