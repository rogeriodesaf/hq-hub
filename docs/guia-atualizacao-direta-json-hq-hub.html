Guia de atualizacao direta por JSON no HQ-HUB

Como abrir um arquivo JSON, entender cada campo e importar a edicao atualizada para o catalogo.

Antes de editar

  1. Faca uma copia do arquivo original antes de alterar.
  2. Edite em um programa que preserve texto puro, como VS Code, Notepad++ ou editor semelhante.
  3. Salve em UTF-8 para evitar textos quebrados, como 1Âª SÃ©rie no lugar de 1ª Série.
  4. Nao coloque comentarios dentro do arquivo. JSON valido nao aceita comentarios.
  5. Mantenha aspas, virgulas, colchetes e chaves exatamente no formato correto.
Se faltar uma virgula, sobrar uma virgula no final de uma lista ou uma aspa ficar aberta, o HQ-HUB nao consegue importar o JSON.

Como ler o JSON

O JSON e organizado em blocos. Cada bloco guarda uma parte da edicao.

{
  "origem": { ... },
  "serieBrasileira": { ... },
  "totalEdicoes": 1,
  "totalHistorias": 5,
  "avisos": [],
  "edicoes": [
    {
      "numero": "12",
      "historias": [
        { "ordem": 1, "publicacaoOriginal": { ... } }
      ]
    }
  ]
}

Quando o campo estiver entre aspas, e texto. Quando nao estiver entre aspas, pode ser numero, lista ou bloco.

Bloco origem

Guarda informacoes sobre de onde veio o arquivo ou quais fontes foram usadas.

Campo O que significa Como preencher
"origem": { Inicio do bloco de origem. Nao altere a estrutura. Edite apenas os campos dentro dele.
"arquivoEntrada" Nome do arquivo usado como entrada. Opcional. Pode usar algo como a-saga-dos-x-men-12.json.
"url" Fonte principal consultada. Use a URL da pagina da edicao, Amazon, Panini, Guia dos Quadrinhos ou outra fonte confiavel.
"urlsProcessadas" Lista de URLs usadas para montar o arquivo. Pode repetir a URL principal ou incluir outras fontes. Cada URL precisa ficar entre aspas.
"geradoEm" Data em que o arquivo foi gerado. Use o formato AAAA-MM-DD, por exemplo 2026-06-30.
"gerador" Nome de quem ou do processo que gerou o JSON. Pode ser HQ-HUB, HQ-HUB robo Guia dos Quadrinhos ou nome interno combinado.
} Fim do bloco de origem. Nao remova.

Bloco serieBrasileira

Identifica a serie brasileira onde a edicao sera criada ou atualizada.

Campo O que significa Como preencher
"serieBrasileira": { Inicio dos dados da serie no Brasil. Nao altere a estrutura.
"titulo" Nome da serie brasileira. Exemplo: A Saga dos X-Men.
"fase" Fase, linha editorial ou temporada. Exemplo: 1ª Série. Corrija textos com encoding quebrado antes de importar.
"editora" Editora brasileira. Exemplo: Panini, Abril, Mythos.
"volume" Volume ou temporada da serie. Use numero sem aspas. Exemplo: 1. Muito importante para separar V1, V2 e novas fases com o mesmo titulo.
} Fim do bloco da serie brasileira. Nao remova.

Totais e avisos

Campo O que significa Como preencher
"totalEdicoes" Quantidade de edicoes dentro do arquivo. Se o arquivo tem uma edicao dentro de edicoes, use 1. Se tiver duas, use 2.
"totalHistorias" Quantidade total de historias somando todas as edicoes do arquivo. Conte todos os itens dentro de historias.
"avisos" Lista de observacoes geradas no preparo do arquivo. Normalmente pode ficar vazio: []. Se houver aviso, use lista de textos.

Bloco edicoes

"edicoes": [ inicia a lista de edicoes que serao importadas. Cada edicao fica dentro de chaves { ... }. Se houver mais de uma edicao, separe os blocos por virgula.

Campo O que significa Como preencher
"numero" Numero da edicao brasileira. Use texto entre aspas para aceitar numeros especiais, como "12", "1A" ou "Especial".
"tituloChamada" Titulo de capa ou chamada da edicao. Pode ficar vazio com "" quando nao houver titulo de chamada.
"dataPublicacao" Data numerica da publicacao. Use AAAA-MM-DD. Se souber apenas mes e ano, escolha uma data padrao combinada e use publicadoTexto para o texto real.
"publicadoTexto" Texto de publicacao mostrado para o usuario. Exemplo: maio de 2023.
"editora" Editora da edicao. Geralmente igual a editora da serie, por exemplo Panini.
"licenciador" Dono/licenciador original do material. Exemplo: Marvel Comics, DC, Image.
"categoria" Tipo editorial da publicacao. Exemplo: Edição Especial, Minissérie, Revista.
"genero" Genero principal. Exemplo: Super-heróis, Terror, Humor.
"status" Status editorial da serie/edicao. Exemplo: Título encerrado ou Em circulação.
"numeroPaginas" Total de paginas da edicao brasileira. Use numero sem aspas, por exemplo 148.
"formato" Formato fisico da publicacao. Pode ter quebra de linha usando \n, como Americano (17 x 26 cm)\nColorido/Lombada quadrada.
"precoCapa" Preco de capa. Use ponto para decimal: 39.9. Nao use 39,90 dentro do JSON.
"urlCapa" URL da imagem da capa brasileira. Use imagem que carregue externamente, de preferencia Amazon, Panini ou fonte confiavel. Evite URL direta de imagem do Guia dos Quadrinhos.
"descricao" Sinopse ou resumo da edicao brasileira. Pode ser texto longo. Se tiver aspas dentro do texto, elas precisam ser escapadas ou removidas.

Bloco historias

"historias": [ inicia a lista de historias publicadas dentro daquela edicao. Cada historia tem seus proprios campos e um bloco de publicacao original.

Campo O que significa Como preencher
"ordem" Posicao da historia dentro da edicao. Use numero sem aspas: 1, 2, 3.
"tituloPortugues" Titulo publicado em portugues. Campo importante para o sistema reconhecer a historia. Se nao houver titulo, use sem titulo.
"tituloOriginal" Titulo original da historia. Pode ficar vazio se nao souber.
"quantidadePaginas" Quantidade de paginas daquela historia. Use numero sem aspas.
"resumo" Resumo, creditos ou observacoes da historia. Pode ser vazio. Para quebrar linha dentro do texto, use \n.

Bloco publicacaoOriginal

Fica dentro de cada historia e informa em qual revista original aquela historia saiu.

Campo O que significa Como preencher
"publicacaoOriginal": { Inicio dos dados da publicacao original. Deixe dentro da historia correta.
"serieOriginal" Serie original estrangeira. Exemplo: Uncanny X-Men, The (1963). Campo essencial para criar vinculo.
"numeroOriginal" Numero original estrangeiro. Use texto entre aspas, por exemplo "200". Campo essencial para criar vinculo.
"anoOriginal" Ano da publicacao original. Use numero sem aspas, por exemplo 1985.
"texto" Descricao pronta da publicacao original. Exemplo: Uncanny X-Men, The (1963) n° 200/1985 - Marvel Comics.
"urlOrigem" URL da pagina da publicacao original. Pode ser Guia dos Quadrinhos, Marvel, DC Database ou outra fonte.
"urlCapa" URL da capa da publicacao original. Use uma imagem que carregue externamente. Pode ficar vazio se nao tiver fonte confiavel.
"urlCompraAmazon" Link de compra, quando existir. Opcional. Pode ficar vazio com "".
} Fim do bloco da publicacao original. Nao remova.

Linhas de abertura e fechamento

As chaves e colchetes sao tao importantes quanto os campos.

Simbolo Funcao Cuidado
{ e } Abrem e fecham um objeto, como uma edicao, historia ou origem. Toda chave aberta precisa ser fechada.
[ e ] Abrem e fecham uma lista, como edicoes, historias ou urlsProcessadas. Itens dentro da lista ficam separados por virgula.
, Separa campos ou itens de lista. Nao coloque virgula depois do ultimo campo de um bloco.
"" Representa texto vazio. Pode ser usado em campos opcionais.
[] Representa lista vazia. Bom para avisos quando nao houver avisos.

Exemplo de uma historia completa

{
  "ordem": 1,
  "tituloPortugues": "O julgamento de Magneto!",
  "tituloOriginal": "The Trial of Magneto!",
  "quantidadePaginas": 40,
  "resumo": "Texto do resumo ou creditos.",
  "publicacaoOriginal": {
    "serieOriginal": "Uncanny X-Men, The (1963)",
    "numeroOriginal": "200",
    "anoOriginal": 1985,
    "texto": "Uncanny X-Men, The (1963) n° 200/1985 - Marvel Comics",
    "urlOrigem": "http://www.guiadosquadrinhos.com/...",
    "urlCapa": "https://cdn.marvel.com/...",
    "urlCompraAmazon": ""
  }
}

Para adicionar outra historia, copie esse bloco, cole depois dele, coloque uma virgula entre os blocos e atualize ordem, titulos, paginas e publicacao original.

Checklist antes de importar

  • O texto esta sem caracteres quebrados, como Ã§, Ã£ e Âª no lugar errado.
  • serieBrasileira.titulo, fase, editora e volume identificam a serie correta.
  • numero identifica a edicao correta.
  • urlCapa carrega fora do site de origem.
  • totalEdicoes bate com a quantidade de itens em edicoes.
  • totalHistorias bate com a quantidade de itens em historias.
  • Cada historia tem tituloPortugues, serieOriginal e numeroOriginal sempre que possivel.
  • O JSON foi validado antes de importar.

Como importar depois de editar direto

  1. Abra a tela de Importacao do HQ-HUB.
  2. Cole o JSON editado no campo de texto do JSON.
  3. Se quiser conferir no formulario, use a opcao de carregar o formulario pelo JSON.
  4. Clique em Importar para o catalogo.
  5. Confira a mensagem final com edicoes criadas, edicoes atualizadas, historias criadas e vinculos criados.
Quando a intencao for apenas corrigir uma edicao ja existente, mantenha o mesmo titulo, volume e numero. Assim o HQ-HUB entende que deve atualizar aquela edicao em vez de criar outra serie parecida.
